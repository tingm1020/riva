step 2解決問題
Q1 客製輪播：圖片與文字的配合？回到初始值？
一開始接到的需求是希望以 GIF 配合文字輪播，但每一次滑動到下一個 slide 就必須從 step1 開始，GIF 是無法清空重來的。
而後改成了圖片與文字皆以 setInterval 的方式去寫。但這是我第一次使用 setInterval ，為了清除 clearInterval 花了一點時間研究。
而後我發現，原來每次重叫一次這個計時器，就必須 clearInterval 一次，並且要額外寫一個function去停止這個計時器，並定義當輪播不等於初始值時，應該回到初始值，意即if ( num >= 0 ) { num = 0; };
Q2 如何讓漸層色適應各瀏覽器？
若文字需使用漸層色，應事先考量IE瀏覽器的支援性：事先用@media screen\0去針對舊版IE的漸層文字覆寫為純色，避免瀏覽時出現異常。
Q3 如何平均每個產品的視覺比例？
一開始在切圖時，就需找出最大高度與最大寬度，讓每個產品圖都在同個比例以 width:100% 縮放。
另外，在所有物件都要求齊高的前提下，需計算一定的高度去撐開物件，讓使用者在縮放視窗時，物件的排列有一致的整齊度。
Q4 放置GIF為何會有白邊？
GIF應避免原圖片有中空、遮罩等效果，在對比色較高的頁面下，易有白色毛邊。

Q3 viewport 定義固定寬度的優缺點？
在轉接到 RWD 的時期，會以 viewport 固定寬來減少手機版破版的可能。優點是不同裝置的手機都可以顯示設計最理想的排版狀態、一些特殊且較複雜的頁面可以避免跑版，缺點是當桌上裝置縮放到小視窗時，版面會有破版狀況。
Final呈現
因為該專案是一張比較高級別的信用卡，因此在視覺體驗上都希望可以有緩慢層次的呈現。因此每滾動一屏，都會有 4 秒的 opacity 的運算呈現出頁面。

step 1架構思考
KV文字

因KV與其他視覺製作物同源，因此在處理上需更謹慎。我會以vw去寫字級，避免px在畫面縮小時候可能有比例不均的問題。

按鈕樣式

找出所有樣式的共通點去寫一個主要樣式，再以不同的元素去區別不同class，避免重複撰寫。

全屏展開

手機版在menu展開時，最底下的項目會有被瀏覽器預設的menu bar遮蔽，可考慮以height:90vh去解決。

step 2解決問題
Q1 物件高度的定義？
我常用的方式是用 padding 去推出物件的高。但有可能會遇到同物件下文字量不足撐到同高的問題，這時候就會以min-height去做高度的統一。
Q2 貼心的展開設計
如果有必須展開的物件，勢必頁面會被往下延展。這時候可以在點擊展開的物件上觸發滾動，把使用者整體的視線一起轉移到展開的頁面。
Q3 注意事項的 text-indent
整齊劃一的注意事項文字，除了以 list-style-type 去做整理外，有些非網頁上原始的樣式要對齊的話，就會以 text-indent + margin-left 去整理。這麼一來繁瑣的文字至少有版面對齊，不會令使用者感到煩躁。
Final呈現
這個專案每一個區塊的原始設計都是一屏的高度，因此在設計上會有背景圖的運用。
相對要注意的就會是每一區塊高度和間距必須平均。

在圖片延展性有限的狀況去限區域高度

  • 在KV的處理上，我不會特別去侷限高度，避免裁切掉重要部分，且圖片皆以視窗大小 width:100% 去縮放。
    但在其他內頁就會有背景圖片的寬高限制，我就會直接定下 height ，並再用一個區塊包住所有內容限定 max-width 不再讓它延展，讓畫面維持在最適切的範圍。但此處也要注意背景圖片中，設計希望不要被切到的地方，去調整 background-position。
